# Quartz 前端研发任务清单

## ✅ 已完成

- [x] **全文检索优化**：支持复合检索（同时包含多个标签 `#` 或属性 `@`）；添加 `-` 符号支持，用于排除指定内容

## 🔄 当前优先级（建议顺序）

### P0 - 基础体验优化
- [x] **构建缓存问题**：新增页面后图谱没有自动重绘 - 已解决 ✅

### P1 - 图谱核心功能
- [x] **图谱-边显示优化**：
  - [x] 增加有向边箭头（`showArrows` 配置，默认开启）
  - [ ] 增加记录边名称（追溯边的来源类型，显示正文引用或 YAML 属性）
  
- [ ] **图谱-构建时优化**：
  - 构建时预计算每个页面的局部图谱关联节点
  - 按节点 ID 拆分为多个 JSON 文件，减少运行时运算量
  - 考虑分片策略（按首字母或子网络）

### P2 - 图谱高级功能
- [ ] **图谱-聚合节点**：
  - 按指定聚合字段（属性、标签或日期范围）聚合同类节点
  - 显示每个聚合节点包含的子节点数量
  - 聚合节点可以展开
  
- [ ] **图谱-追踪展示**：
  1. 关联节点深度/广度控制：默认展示直接相关的 m 个节点、间接相关的 n 层节点（m、n 可配置）
  2. 关联追溯：用户可以不停地点击追溯关联关系
  3. 详情列表组件：图谱下方增加列表区域，展示与当前节点直接相关的节点详情

### P3 - 目录与链接优化
- [ ] **反向链接优化**：
  1. 支持根据 U3 的配置调整排序
  2. 按文件夹、聚合字段分组显示反向链接，可收起展开
  3. 支持分批加载/虚拟滚动以优化长列表渲染
  
- [ ] **目录与文件夹页优化**：
  - 目录：支持根据 U3 的配置调整排序
  - 文件夹页：支持根据 U3 的配置调整排序；支持分批加载/虚拟滚动

## 💡 技术实现建议

- 图谱功能建议单独制作页面，支持与 U3 系统内功能嵌套关联使用
- 通过 API 接受图谱 JSON 数据，便于与 U3 集成
