### 2. 虚拟节点处理修复 ✅

**问题**：
- 虚拟节点显示双向箭头
- 依赖 `virtualNodeIndex.json` 导致链接方向错误

**解决方案**：
- 像 tags 一样**动态计算**虚拟节点
- 虚拟节点永远是被指向的目标（实体节点 → 虚拟节点）
- 使用 `"source->target"` key 去重

**链接方向修正**：
```
修改前: 虚拟节点 ──> 笔记（错误）
修改后: 笔记 ──> 虚拟节点（正确）
```

### 3. 边名称显示 ✅

**实现内容**：
- 区分链接来源类型：正文引用 vs YAML frontmatter 属性
- `SourcedLink` 类型：记录链接目标和来源 `{ type: 'content' | 'frontmatter', field?: string }`
- 构建时存储 `source_field` 到数据库
- 前端在边的中点显示来源标签（hover 时显示）

**技术方案**：
1. `links.ts` 的 `CrawlLinks` 插件区分正文链接和 frontmatter 链接
2. `contentIndex` 导出 `sourcedLinks` 供前端使用
3. 图谱渲染时在中点位置显示标签（"正文" 或属性名）

**显示效果**：
```
节点 A ──[正文]──> 节点 B
节点 A ──[author]──> 节点 C  (YAML 属性)
```
