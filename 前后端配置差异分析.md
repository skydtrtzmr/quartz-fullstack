# 前后端配置差异分析

## 概述

本文档对比分析 Server (Go) 和 Client (Quartz/TypeScript) 的配置体系。

---

## Settings 目录结构

```
settings/{domain}/
├── quartz.config.json   # 配置覆盖（baseUrl, pageTitle, graph）
├── quartz.layout.json    # 布局配置（backlinks, explorer, aggregation）
└── .quartz-cache.db     # SQLite 图谱数据库（自动创建）
```

### 配置映射

| 概念 | 来源 |
|------|------|
| Domain ID | settings 子目录名 |
| Display Name | `quartz.config.json` 的 `pageTitle` |
| Base URL | `quartz.config.json` 的 `baseUrl` |
| Graph 配置 | `quartz.config.json` 的 `graph` |
| Layout 配置 | `quartz.layout.json` |

---

## quartz.config.json

```json
{
  "baseUrl": "127.0.0.1:8767/testwork0",
  "pageTitle": "testwork0",
  "graph": {
    "tags": {
      "color": "#4a9eff",
      "displayName": "标签"
    }
  }
}
```

## quartz.layout.json

```json
{
  "backlinks": {
    "hideWhenEmpty": false,
    "aggregation": {
      "folder": {
        "depth": 1,
        "flatten": true
      },
      "fields": [
        { "field": "date", "granularity": "year", "order": 1 }
      ]
    }
  }
}
```

---

## Server vs Client 职责

| 角色 | 职责 |
|------|------|
| Server | CRUD `quartz.config.json` 和 `quartz.layout.json` |
| Client | 构建时读取 settings 目录，deepMerge 到 config/layout |
