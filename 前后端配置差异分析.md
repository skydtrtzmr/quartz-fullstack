# 前后端配置差异分析

## 概述

本文档对比分析 Server (Go) 和 Client (Quartz/TypeScript) 的配置体系，列出差异及需要调整的部分。

---

## 1. Settings 目录结构对比

### 当前 Client 使用的文件

```
settings/{domain}/
├── config.json          # Quartz 配置覆盖（baseUrl, theme 等）
├── layout.json          # 布局配置（explorer, backlinks, folderPage）
└── .quartz-cache.db     # SQLite 图谱数据库（自动创建）
```

### Server domain_config.go 生成的文件

```
settings/{domain}/
├── domain_config.json   # 业务域完整配置（后端管理）
└── config.json          # 从 domain_config 转换而来（供 Client 使用）
```

### 差异点

| 项目 | Client | Server | 说明 |
|------|--------|--------|------|
| 配置入口 | `config.json` | `domain_config.json` | Server 新增 domain_config.json 作为主配置 |
| 配置转换 | 无 | `ApplyToQuartzConfig()` | Server 自动生成 Client 需要的 config.json |
| layout.json | 手动维护 | 未涉及 | Server 暂未管理 layout.json |

---

## 2. 配置格式对比

### Client config.json 格式

```json
{
  "baseUrl": "127.0.0.1:8767/xm",
  "graph": {
    "tags": {
      "color": "#4a9eff",
      "displayName": "标签"
    }
  }
}
```

### Server DomainConfig 格式

```go
type DomainConfig struct {
    DomainName      string              // 业务域标识
    DisplayName     string              // 显示名称
    Description     string              // 描述
    RootFolders     []RootFolderConfig  // 一级目录配置（U3 配置）
    AggregationFields AggregationConfig // 聚合字段（图谱/目录）
    BuildOverrides  BuildConfig         // 构建设置覆盖
}
```

生成的 config.json：
```json
{
  "baseUrl": "127.0.0.1:8767/xm",
  "graph": {
    "tags": {
      "displayName": "标签",
      "color": "#4a9eff"
    }
  }
}
```

### 差异点

| 配置项 | Client 支持 | Server 支持 | 状态 |
|--------|------------|-------------|------|
| baseUrl | ✅ | ✅ | 一致 |
| theme | ✅ | ⚠️ 部分 | Server 未覆盖 theme |
| ignorePatterns | ✅ | ❌ | Server 未覆盖 |
| locale | ✅ | ❌ | Server 未覆盖 |
| graph.tags | ✅ | ✅ | 一致 |
| graph.category | ✅ | ✅ | Server 已支持 |
| enableSPA | ✅ | ❌ | Server 未覆盖 |
| enablePopovers | ✅ | ❌ | Server 未覆盖 |

## 3. 需要调整的前端 Client 部分

### 3.1 layout.json 支持（Server 需增强）

当前 `quartz.layout.ts` 从 `--settings` 目录加载 `layout.json`：

```typescript
const layoutJsonPath = path.join(settingsPath, "layout.json")
```

**Server 现状**：未生成 layout.json

**建议调整**：
1. Server 在 `ApplyToQuartzConfig()` 中增加 layout.json 生成
2. 或在 DomainConfig 中增加 layout 字段

```go
// Server DomainConfig 增加
type DomainConfig struct {
    // ... 现有字段
    Layout LayoutOverrides `json:"layout"`
}

type LayoutOverrides struct {
    Explorer   *ExplorerConfig   `json:"explorer,omitempty"`
    Backlinks  *BacklinksConfig  `json:"backlinks,omitempty"`
    FolderPage *FolderPageConfig `json:"folderPage,omitempty"`
}
```

### 3.2 更多 configuration 字段覆盖

当前 `quartz.config.ts` 支持的 configuration 字段：

```typescript
interface QuartzConfig {
  configuration: {
    pageTitle: string
    pageTitleSuffix: string
    enableSPA: boolean
    enablePopovers: boolean
    analytics: object
    locale: string
    baseUrl: string
    ignorePatterns: string[]
    defaultDateType: string
    theme: {
      fontOrigin: string
      cdnCaching: boolean
      typography: object
      colors: object
    }
  }
}
```

**Server 当前生成**：仅 `baseUrl` 和 `graph`

**建议调整 Server**：在 `generateQuartzConfig()` 中增加更多字段：

```go
func (dm *DomainManager) generateQuartzConfig(domainConfig *DomainConfig) map[string]interface{} {
    config := map[string]interface{}{
        "baseUrl": domainConfig.BuildOverrides.BaseUrl,
        "pageTitle": domainConfig.DisplayName,  // 新增
        "locale": "zh-CN",                       // 新增
        "enableSPA": domainConfig.BuildOverrides.EnableSPA,  // 新增
        "enablePopovers": domainConfig.BuildOverrides.EnablePopovers,  // 新增
    }
    // ...
}
```

### 3.3 路径解析差异

**Client 行为**：
- `--settings={path}` 指定配置目录
- 读取 `{path}/config.json` 和 `{path}/layout.json`
- SQLite 数据库路径：`{path}/.quartz-cache.db`

**Server 行为**：
- `cfg.SettingsDir` 是根目录
- 业务域配置目录：`{SettingsDir}/{domain}/`
- 自动生成 `{SettingsDir}/{domain}/config.json`

**当前匹配情况**：✅ 一致

Server 启动命令：
```
--settings=E:/.../settings/xm
```

对应 Client 会加载：
- `settings/xm/config.json` ✅（Server 生成）
- `settings/xm/layout.json` ⚠️（需手动维护或 Server 增强）
- `settings/xm/.quartz-cache.db` ✅（Client 自动创建）

## 4. 聚合字段配置差异

### Server AggregationConfig

```go
type AggregationConfig struct {
    GraphFields    []GraphFieldMapping
    ExplorerFields []ExplorerFieldMapping
}
```

### Client 实际使用

**Graph 组件**：`quartz/components/scripts/graph.inline.ts`
- 使用 `contentIndex.json` 中的数据
- 未直接使用 `config.json` 中的 graph 字段

**Explorer 组件**：`quartz/components/scripts/explorer2.inline.ts`
- 使用 `data-data-fns` 属性传递配置
- 通过 `quartz.layout.ts` 中的 `Explorer2()` 组件配置

### 差异点

Server 的 `aggregation_fields` 概念在 Client 中没有直接对应：

1. **GraphFields**：Server 生成到 `config.json.graph`，但 Client Graph 组件可能不读取
2. **ExplorerFields**：Server 配置未传递到 Client Explorer

**建议调整 Client**：

修改 `Explorer2.tsx` 支持从 `config.json` 读取聚合字段配置：

```typescript
// quartz/components/Explorer2.tsx
// 增加读取 config.json 中 explorer 配置的逻辑
```

或在 `quartz.layout.ts` 中读取 `layout.json` 时支持聚合字段：

```typescript
// quartz.layout.ts
Component.Explorer2({
  stickyHeaders: layoutCfg.explorer?.stickyHeaders ?? false,
  // 新增：从 layout.json 读取聚合字段配置
  groupBy: layoutCfg.explorer?.groupBy,
})
```

## 5. RootFolders 配置（U3 配置）

### Server RootFolderConfig

```go
type RootFolderConfig struct {
    Name        string // 目录名
    DisplayName string // 显示名
    Description string // 描述
    Icon        string // 图标
    Order       int    // 排序
    Visible     bool   // 是否可见
}
```

### Client 现状

Client 目录结构直接从文件系统读取，没有 "RootFolders" 配置概念。

### 差异点

Server 支持配置一级目录的显示属性，但 Client 没有对应实现。

**建议调整 Client**：

方案1：在 `quartz.layout.ts` 中增加 RootFolders 配置

```typescript
Component.Explorer2({
  rootFolders: layoutCfg.rootFolders ?? [],  // 从 layout.json 读取
})
```

方案2：Server 将 RootFolders 写入 layout.json，Client 读取并过滤目录

---

## 6. 总结：需要前端调整的部分

### 高优先级

1. **layout.json 增强**（Server 或 Client）
   - 当前：layout.json 需手动维护
   - 建议：Server 的 DomainConfig 增加 layout 字段，自动生成 layout.json

2. **更多 configuration 字段**（Server 增强）
   - 当前：Server 只生成 baseUrl
   - 建议：Server 生成 pageTitle, locale, enableSPA 等常用字段

### 中优先级

3. **聚合字段配置同步**（Client 调整）
   - 当前：Server 配置 AggregationFields 但 Client 未使用
   - 建议：Client Explorer2 组件支持从 config.json 读取聚合配置

4. **RootFolders 支持**（Client 调整）
   - 当前：Server 配置 RootFolders 但 Client 未使用
   - 建议：Client Explorer2 组件支持根目录过滤/排序配置

### 低优先级

5. **theme 配置**（Server 增强）
   - 当前：theme 在 quartz.config.ts 中硬编码
   - 建议：Server DomainConfig.BuildOverrides 增加 theme 字段

---

## 7. 建议的前端调整方案

### 方案 A：Client 增强 layout.json 支持

修改 `quartz.layout.ts`：

```typescript
interface LayoutConfig {
  explorer?: {
    sortBy?: SortByConfig
    groupBy?: string        // 新增：聚合字段
    rootFolders?: RootFolderConfig[]  // 新增：根目录配置
  }
  backlinks?: {...}
  folderPage?: {...}
}

// 在组件中使用
Component.Explorer2({
  stickyHeaders: false,
  sortFn: createSortFn(layoutCfg.explorer?.sortBy),
  groupBy: layoutCfg.explorer?.groupBy,
  rootFolders: layoutCfg.explorer?.rootFolders,
})
```

### 方案 B：Server 增强 layout.json 生成

修改 `server/domain_config.go`：

```go
func (dm *DomainManager) ApplyToQuartzConfig(domain string) error {
    // ... 现有 config.json 生成
    
    // 新增：生成 layout.json
    layoutConfig := map[string]interface{}{
        "explorer": map[string]interface{}{
            "sortBy": config.AggregationFields.ExplorerFields[0],
            "groupBy": config.AggregationFields.ExplorerFields[0].FieldName,
            "rootFolders": config.RootFolders,
        },
    }
    // 写入 layout.json
}
```
